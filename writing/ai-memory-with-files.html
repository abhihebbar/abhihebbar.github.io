The AI isn't the problem. The files are.

March 2026 · AI

I've been running OpenClaw for a few months — it's a personal AI setup that runs on a server, connects to Telegram, and can do things like send emails, write and commit code, manage files. Not a chatbot you open occasionally. Something that's always on.

The capability surprised me. The inconsistency frustrated me more.

Sometimes it remembers a preference I mentioned two days ago. Sometimes it contradicts something we settled three messages back. No pattern to it. Just enough randomness to make it hard to rely on.

What I eventually figured out is that the inconsistency isn't really the AI's fault. It's an information problem.

The model reads a set of files at the start of each session. What it knows about you comes from those files — not from some persistent memory layer, not from conversation history. Just files. When those files are clean and current, the sessions are good. When they're messy, outdated, or contradictory, the sessions reflect that.

The file structure I landed on

There are five files that matter:

MEMORY.md — the long-term facts. Who I am, how I work, what my projects are, shortcut conventions. Kept deliberately short. The moment it bloats with task-specific detail, it stops working well.

SOUL.md — who the AI is. Personality, working style, how it should behave. The AI can edit this itself when it learns something about how I like to work.

USER.md — a short note about me for context. Name, timezone, how I want responses. One page max.

memory/YYYY-MM-DD.md — daily logs. Raw notes from each session. What happened, what was decided, what to follow up on. The AI writes these, not me.

FOLLOWUPS.md — things I need to do. The AI checks this and nudges me if something needs attention.

The whole thing is maybe 10 files. Plain markdown on the same server.

What breaks it

Duplication. The same thing written slightly differently in two places — the AI picks one and ignores the other, and you never know which.

Staleness. Instructions that made sense three weeks ago but no longer apply. The AI has no way to know they're outdated.

Mixing concerns. Task-specific detail (how a particular project works, what tools it uses) sitting in the general memory file alongside personal preferences. I pull task detail into separate project folders now. MEMORY.md is only for things that apply to every session.

The prompts I use to maintain it

To save something to memory:

++ remember that [thing]

To update the personality file:

>> [update to how I want you to behave]

When a session goes wrong, instead of just moving on:

Something went wrong in that last session. What context were you missing? What should be in the files so this doesn't happen again?

That last one is the most useful. The AI will usually tell you exactly what was missing — and then you can write it down properly.

What changed

The current setup is three days old. Before this, I'd installed and uninstalled twice — sessions were too chaotic to be useful.

The AI didn't change. The files did.

I'm still figuring out the right things to keep versus discard. But the basic shape works: short files, each doing one job, maintained as you go. Treat it like a handoff document you're leaving for yourself.
